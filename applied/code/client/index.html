The rag has memory of chat upto last 5 messages, so follow-ups can be asked. To start a new session click on the new button on the left. The response will have subcontent showing (in this order): Urgency Level, Confidence, Overconfidence, and Sources
I built this using langchain, chromadb, voyage-ai embeddings & reranking, and gemini (2.5-flash) llm Trained using medquad, WHO guidelines, NHS guidelines, NICE guidelines, CDC Guide Works best on questions about cardiac health, fever, sprains, uti, diarrhoea, constipation, common cold and basic emergencies

Some improvements/learnings from last time:

  • Reranking:

    I implemented a reranker as specified in the task guide, and I really liked the way it works. While chroma used stuff like cosine and distance b/w vectors to find similarity, rerankers use cross-encoders based on the text allowing for higher accuracy and it related with what I've been learning in base ml so I found it very cool.

  • Memory:

    This is what I liked most. Last time I implemented no memory and just called an llm. This time, I stored the last 6-7 chats in-memory, and summarized them along with the query with an llm. I then passed that into chroma for retrieval and the same into the prompt for my final llm. I really liked the way I could ask it follow-ups.

  • UI Design:

    I was dissapointed by the way my task last time had a textbox, submit and response only. So this time, I wanted to make it look like a proper chat app. This was one of the hardest parts. I kept changing small things here and there and it took a long long time for me to style it the way I thought looked good. The javascript part felt natural and similar to python/other languages so I was able to get that done.

  • Collecting Data:

    I learnt basic web-scraping and utilizing online tools to convert data from one form to another, such that I can use them in the best way for my rag. I learnt a lot about different methods of data sources and different online tools that can convert data for me.

  • Code Design:

    I'm not sure if this even is the right term, but what I mean is, last time my code was just a bundle of functions in random files, and I didn't design it properly and that led to me ingesting and embedding multiple times. This time, I tried to make it so that the code was reusable, and I researched proper coding styles and I think I've achieved something better than last time.
